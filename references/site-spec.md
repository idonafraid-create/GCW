# SITE_SPEC contract

Every task creates `.gcw/SITE_SPEC.md` as the unique implementation specification. Keep all 12 numbered sections from the supplied template; write `N/A` rather than deleting an inapplicable section.

Fixed evidence is `.gcw/evidence/site-inventory.json`, `route-map.json`, `interaction-states.json`, desktop/mobile screenshots, and `network/`. Add a full route map for multiple page families, render/input/output models for GPU surfaces, state timelines for complex motion, an asset inventory for asset-heavy work, Design DNA before creative rebuild, and provenance/hashes/recovery strategy for recovery work.

Record each subsystem as `Exact`, `Approximate`, `Unknown`, or `Excluded`, with evidence and acceptance criteria. Copy final results into `CLONE_REPORT.md`.
